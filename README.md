# Functional Modeling of Systems — Система помощи мигрантам

## Описание проекта
REST API для формирования персональной **дорожной карты** мигранта. Оператор (ADMIN) задаёт
правила с условиями, пользователь заполняет профиль мигранта и получает список шагов
(что нужно получить/сделать) с дедлайнами, рассчитанными по действующим правилам.

Основные возможности:
- регистрация пользователей и просмотр профиля;
- создание и управление профилем мигранта (данные о себе);
- управление правилами формирования карты и их условиями (только для ADMIN);
- генерация дорожной карты по действующим правилам и данным мигранта;
- справочники стран и целей визита.

## Технологии
- **Java 17**
- **Spring Boot**
- **Spring Web**
- **Spring Data JPA**
- **Spring Security** (form login, BCrypt, роли USER/ADMIN)
- **Lombok**
- **Swagger (OpenAPI 3)**
- **PostgreSQL**

## Запуск проекта
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/GARFIELD-CAT/functional-modeling-of-systems
   ```
2. Перейдите в директорию проекта:
   ```bash
   cd functional-modeling-of-systems
   ```
3. Настройте подключение к PostgreSQL в `src/main/resources/application.properties`
   (URL базы, имя пользователя, пароль).
4. Запустите приложение:
   ```bash
   ./mvnw spring-boot:run
   ```

## Аутентификация и доступ
API защищён Spring Security (роли **USER** и **ADMIN**). Доступ к ручкам:

| Ресурс | Доступ |
| --- | --- |
| `POST /api/users` (регистрация) | Открыт для всех |
| `GET /api/users/me` | Любой аутентифицированный |
| `GET /api/users/{id}` | ADMIN |
| `/api/migrants/**` | USER или ADMIN |
| `POST /api/road-maps` | USER или ADMIN |
| `GET /api/countries`, `GET /api/purposes-of-visit` | Открыт для всех |
| `/admin/rules/**` | ADMIN |
| Swagger UI / OpenAPI | Открыт для всех |

Профиль мигранта доступен только его владельцу или администратору (проверка владельца).

## Эндпоинты

### Пользователи
#### Создание нового пользователя (регистрация)
- **URL:** `/api/users`
- **Метод:** `POST`
- **Доступ:** открыт для всех
- **Тело запроса:**
  ```json
  {
    "login": "string (8–30 символов)",
    "password": "string (8–30 символов)"
  }
  ```
- **Ответ:**
    - `201 Created` — пользователь успешно создан (возвращается `{ "id", "login" }`)
    - `400 Bad Request` — некорректные данные
    - `409 Conflict` — пользователь с таким логином уже существует

#### Получение пользователя по ID
- **URL:** `/api/users/{id}`
- **Метод:** `GET`
- **Доступ:** ADMIN
- **Ответ:**
    - `200 OK` — пользователь найден
    - `404 Not Found` — пользователь не найден

#### Получение текущего пользователя
- **URL:** `/api/users/me`
- **Метод:** `GET`
- **Доступ:** любой аутентифицированный
- **Ответ:**
    - `200 OK` — профиль текущего пользователя
    - `401 Unauthorized` — пользователь не аутентифицирован

### Профили мигрантов
#### Создание профиля мигранта
- **URL:** `/api/migrants`
- **Метод:** `POST`
- **Доступ:** USER или ADMIN (профиль привязывается к текущему пользователю)
- **Тело запроса:**
  ```json
  {
    "entryDate": "yyyy-mm-dd",
    "plannedDurationOfStay": "integer (> 0)",
    "countryOfCitizenshipId": "integer",
    "purposeOfVisitId": "integer",
    "hqsOrFamilyMember": false,
    "hasFingerprinting": false,
    "hasPhotoRegistration": false,
    "hasMigrationRegistration": false,
    "hasMedicalInsurance": false,
    "hasMedicalExamination": false,
    "hasRussianLanguageCert": false,
    "hasInn": false,
    "resettlementParticipant": false,
    "hasUssrCertificate": false,
    "hasRussianDiploma": false
  }
  ```
- **Ответ:**
    - `201 Created` — профиль успешно создан
    - `400 Bad Request` — некорректные данные
    - `404 Not Found` — указанная страна или цель визита не найдена
    - `409 Conflict` — профиль мигранта для текущего пользователя уже существует

#### Получение профиля мигранта по ID
- **URL:** `/api/migrants/{id}`
- **Метод:** `GET`
- **Доступ:** владелец профиля или ADMIN
- **Ответ:**
    - `200 OK` — профиль найден
    - `403 Forbidden` — доступ к чужому профилю запрещён
    - `404 Not Found` — профиль не найден

#### Обновление профиля мигранта
- **URL:** `/api/migrants`
- **Метод:** `PATCH`
- **Доступ:** владелец профиля или ADMIN
- **Тело запроса:** (передаются только изменяемые поля; `id` обязателен)
  ```json
  {
    "id": "integer",
    "entryDate": "yyyy-mm-dd",
    "plannedDurationOfStay": "integer (> 0)",
    "countryOfCitizenshipId": "integer",
    "purposeOfVisitId": "integer",
    "hqsOrFamilyMember": true,
    "hasMedicalInsurance": true
  }
  ```
- **Ответ:**
    - `200 OK` — профиль успешно обновлён
    - `400 Bad Request` — некорректные данные
    - `403 Forbidden` — доступ к чужому профилю запрещён
    - `404 Not Found` — профиль не найден

#### Удаление профиля мигранта по ID
- **URL:** `/api/migrants/{id}`
- **Метод:** `DELETE`
- **Доступ:** владелец профиля или ADMIN
- **Ответ:**
    - `204 No Content` — профиль успешно удалён
    - `403 Forbidden` — доступ к чужому профилю запрещён
    - `404 Not Found` — профиль не найден

#### Получение профиля текущего пользователя
- **URL:** `/api/migrants/me`
- **Метод:** `GET`
- **Доступ:** USER или ADMIN
- **Ответ:**
    - `200 OK` — профиль текущего пользователя
    - `404 Not Found` — у текущего пользователя нет профиля мигранта

### Дорожные карты
#### Формирование дорожной карты
- **URL:** `/api/road-maps`
- **Метод:** `POST`
- **Доступ:** USER или ADMIN (карта строится для текущего пользователя, тело запроса не требуется)
- **Ответ:**
    - `201 Created` — дорожная карта сформирована
    - `404 Not Found` — у текущего пользователя нет профиля мигранта
- **Пример ответа:**
  ```json
  {
    "steps": [
      {
        "title": "string",
        "deadline": "yyyy-mm-dd",
        "message": "Что нужно получить: ...\nЧто нужно сделать: ..."
      }
    ]
  }
  ```
  Если не подошло ни одно правило, карта содержит один шаг с сообщением о невозможности
  её формирования. Если срок по правилу просрочен, в сообщение шага добавляется предупреждение.

### Правила (только для ADMIN)
#### Создание правила с условиями
- **URL:** `/admin/rules`
- **Метод:** `POST`
- **Доступ:** ADMIN
- **Тело запроса:**
  ```json
  {
    "title": "string",
    "description": "string",
    "requiredResult": "string",
    "requiredAction": "string",
    "period": "integer (> 0, срок в днях)",
    "effectiveTo": "yyyy-mm-dd или null (бессрочно)",
    "conditions": [
      {
        "field": "имя поля профиля мигранта, напр. hasMedicalExamination",
        "operator": "EQ | NO_EQ | GT | LT | GTE | LTE | IN | NOT_IN",
        "value": "string, напр. \"false\" или \"1,2,3\" для IN"
      }
    ]
  }
  ```
- **Ответ:**
    - `201 Created` — правило успешно создано
    - `400 Bad Request` — некорректные данные (в т.ч. недопустимое имя поля условия)

#### Получение правила по ID
- **URL:** `/admin/rules/{id}`
- **Метод:** `GET`
- **Доступ:** ADMIN
- **Ответ:**
    - `200 OK` — правило найдено (вместе с условиями)
    - `404 Not Found` — правило с указанным id не найдено

#### Получение списка правил
- **URL:** `/admin/rules`
- **Метод:** `GET`
- **Доступ:** ADMIN
- **Параметры запроса:**
    - `onlyActive` (boolean, по умолчанию `true`) — возвращать только действующие на сегодня правила
- **Ответ:**
    - `200 OK` — список правил (может быть пустым)

#### Обновление правила
- **URL:** `/admin/rules`
- **Метод:** `PATCH`
- **Доступ:** ADMIN
- **Тело запроса:** (передаются только изменяемые поля; `id` обязателен; `conditions` = `null` — условия не трогаем, иначе список заменяется полностью)
  ```json
  {
    "id": "integer",
    "title": "string",
    "period": "integer (> 0)",
    "effectiveTo": "yyyy-mm-dd",
    "conditions": [
      { "field": "string", "operator": "EQ", "value": "string" }
    ]
  }
  ```
- **Ответ:**
    - `200 OK` — правило успешно обновлено
    - `400 Bad Request` — некорректные данные
    - `404 Not Found` — правило с указанным id не найдено

#### Удаление правила
- **URL:** `/admin/rules/{id}`
- **Метод:** `DELETE`
- **Доступ:** ADMIN
- **Ответ:**
    - `204 No Content` — правило удалено (связанные условия удаляются каскадно)
    - `404 Not Found` — правило с указанным id не найдено

## Операторы условий
Условие правила сравнивает поле профиля мигранта (`field`) с ожидаемым значением (`value`)
с помощью оператора (`operator`):

| Оператор | Значение |
| --- | --- |
| `EQ` | равно |
| `NO_EQ` | не равно |
| `GT` | больше |
| `LT` | меньше |
| `GTE` | больше или равно |
| `LTE` | меньше или равно |
| `IN` | входит в список (значения через запятую: `1,2,3`) |
| `NOT_IN` | не входит в список |
